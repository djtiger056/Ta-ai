# 主动消息触发测试指南

## 功能位置

**情绪系统页面** → **主动消息配置** → **立即触发**

## 使用步骤

### 1. 打开配置面板
1. 访问情绪系统页面（`/cerebellum`）
2. 点击右上角的 **"主动消息配置"** 按钮
3. 配置面板会展开

### 2. 配置提示词（可选）
在配置面板中可以设置：
- **默认提示词**：主动消息的基础指引
- **话术模板**：可选的消息灵感（每行一条）
- **主动生图**：是否允许生成图片

### 3. 立即触发测试
1. 点击配置面板右上角的 **"立即触发"** 按钮（闪电图标）
2. 系统会立即生成一条主动消息
3. 消息会发送到 **Web 聊天页面**（`/chat`）
4. 成功后会显示提示：`已触发主动消息：[消息内容]`

### 4. 查看结果
1. 切换到聊天页面（`/chat`）
2. 等待几秒钟（消息会通过轮询机制获取）
3. 主动消息会显示为 **"主动消息"** 标签

## 测试场景

### 场景 1：测试默认提示词
```
默认提示词：你突然想到他了，想发条消息给他。不需要特别的理由，就是想聊聊。
话术模板：（留空）
```
**预期效果**：生成自然的问候消息，如"在干嘛呀"、"想你了"

### 场景 2：测试话术模板
```
默认提示词：你突然想到他了，想发条消息给他。
话术模板：
  今天过得怎么样呀？
  想你了，在干嘛呢？
  有没有想我呀~
```
**预期效果**：生成的消息会参考模板的风格

### 场景 3：测试情绪驱动（需要先触发情绪）
1. 在聊天页面发送正面消息："太开心了！谢谢你！"
2. 观察情绪系统页面，joy 情绪应该上升
3. 当 joy ≥ 0.68 时，系统会自动触发"分享"动机
4. 也可以手动点击"立即触发"测试当前情绪状态下的消息

## 提示词设计建议

### ✅ 好的提示词（自然、不出戏）
```
你突然想到他了，想发条消息给他。
结合你们最近聊过的事情，自然地开口就好。
```

### ❌ 不好的提示词（容易出戏）
```
请以女友身份主动问候用户。
当前情绪状态：喜悦（强度 0.72）
请根据情绪生成合适的消息。
```

**核心原则**：
- 用角色内心独白，不用系统术语
- 强调"自然"、"像平时聊天一样"
- 不要暴露"情绪强度"、"动机类型"等机制

## 常见问题

### Q: 点击"立即触发"后没有反应？
**A**: 检查以下几点：
1. 后端是否正常运行（查看控制台日志）
2. ProactiveChatScheduler 是否已初始化
3. 查看浏览器控制台是否有错误信息

### Q: 消息没有出现在聊天页面？
**A**: 
1. 等待 10 秒（轮询间隔）
2. 刷新聊天页面
3. 检查后端日志是否有发送记录

### Q: 如何测试不同情绪下的消息？
**A**: 
1. 在聊天页面发送不同情感的消息
   - 正面：`太开心了！`、`谢谢你！`
   - 负面：`好难过`、`心情不好`
   - 惊讶：`哇！太惊讶了！`
2. 观察情绪系统页面的情绪变化
3. 点击"立即触发"测试当前情绪状态

### Q: 如何避免消息"出戏"？
**A**: 参考 `message_builder.py` 中的设计：
- 用内心独白代替系统术语
- 用语气引导代替情绪标签
- 保持简短，不过度约束

## 技术细节

### 触发流程
```
点击"立即触发"
  ↓
调用 proactiveApi.triggerOnce()
  ↓
POST /api/proactive/trigger
  ↓
ProactiveChatScheduler.trigger_once()
  ↓
构建提示词（message_builder.py）
  ↓
Bot.generate_proactive_reply()
  ↓
LLM 生成回复
  ↓
发送到 Web 消息队列
  ↓
前端轮询获取（每 10 秒）
  ↓
显示在聊天页面
```

### 提示词构建逻辑
```python
# backend/core/proactive/message_builder.py

1. 基础提示词（default_prompt）
2. 动机内心独白（如果有情绪触发）
3. 语气引导（根据主导情绪）
4. 话术模板参考（随机选择）
5. 格式约束（1-3句话）
```

### 与情绪系统的关系
- **手动触发**：不依赖情绪状态，直接使用 default_prompt
- **情绪触发**：当情绪达到阈值时，会自动添加动机上下文
- **测试建议**：先手动触发测试基础配置，再测试情绪驱动
